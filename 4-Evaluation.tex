We implemented Algorithm~\ref{Alg:solveCHC} in the \golem CHC 
solver~\cite{DBLP:conf/cav/BlichaBS23}; we refer to this implementation as \golemNested. 
We compare \golemNested\ to several state-of-the-art tools: 
\zspacer~\cite{DBLP:journals/fmsd/KomuravelliGC16} (implemented in \textsc{Z3}, v4.13.1), \eldarica~\cite{DBLP:conf/fmcad/HojjatR18} 
(v2.2.0), \abmc~\cite{DBLP:conf/fm/FrohnG24} (v0.7.0), and to existing \golem engines, namely \golemTPA 
and \golemSpacer. 
Evaluation focuses on complex linear CHC instances encoding programs with nested loops. 
Experiments were conducted on Ubuntu 20.04 machine with an AMD EPYC 7452 32-core processor and 8x32 GiB RAM (the artifact is in open access\footnote{\href{https://doi.org/10.5281/zenodo.17055558}
{https://doi.org/10.5281/zenodo.17055558}}). 
Our evaluation addresses the following research questions:
\begin{itemize}
\item \textbf{RQ1:} How does the new algorithm compare to other engines in \golem?
\item \textbf{RQ2:} How does \golemNested perform against state-of-the-art tools?
\end{itemize}

The benchmark suite\footnote{\href{https://github.com/BritikovKI/pretzel-benchmarks/tree/main}
{https://github.com/BritikovKI/pretzel-benchmarks/tree/main}} is composed of nested-loop instances 
selected from the LIA category of CHC-COMP 2024~\cite{DBLP:journals/corr/abs-2404-14923} and 
aeval-benchmarks~\cite{DBLP:conf/cav/FedyukovichPMG19}. 
Additionally, we extracted nested-loop functions from the source code of several tools: \textsc{Z3}~\cite{DBLP:conf/tacas/MouraB08}, 
\textsc{OpenSMT}~\cite{DBLP:conf/sat/HyvarinenMAS16}, \textsc{SeaHorn}~\cite{DBLP:conf/cav/GurfinkelKKN15}, and \golem 
and encoded them as LIA CHC instances using \textsc{Korn}~\cite{DBLP:conf/tacas/Ernst23} and \textsc{SeaHorn}. 
Benchmark selection is based on the presence of nested loops in the original program, 
for the instances without nested loops \golemNested performs similar to \golemTPA (for example on LIA-lin
benchmarks from CHC-Comp which mainly contain single loops, \golemNested is on average 20\% faster
compared to \golemTPA and solves 20 more instances).
The final set contains 130 CHC instances (68 SAT, 62 UNSAT).

%

Our results indicate that \golemNested consistently outperforms both \golemSpacer and \golemTPA for SAT 
and UNSAT instances. 
We attribute this to the modular nature of our algorithm, which decomposes the reachability 
analysis task into multiple, smaller subproblems—thereby reducing the overall complexity of reasoning. 
\golemNested leverages structural information and explicitly analyzes cycles separately, which benefits 
both SAT and UNSAT performance. 
In contrast, \golemTPA merges the entire CHC system into a CHC cycle composed from a single clause, 
losing structural information. 
\golemSpacer, while effective in many scenarios, struggles with deep nested loops. 
In particular, as the number of loop iterations increases, \golemSpacer frequently times out. 
Overall, \golemNested solved 118/130 instances, compared to 78 for \golemTPA and 93 for \golemSpacer. 
To answer \textbf{RQ1}, we conclude that the compositional strategy of \golemNested leads to better performance.
We attribute this to the \emph{modular} summarization-based approach, which uses \tpa for fast deep 
reachability checks and summarization, while maintaining structural information about the system.

Figure~\ref{fig:stateOfArtComp} gives comparison charts for \golemNested to other state-of-the-art CHC solvers. 
\golemNested solves the most benchmarks (118), followed by \eldarica (111), \zspacer (91), and \abmc (55). 
The listed approaches are somewhat orthogonal, as \eldarica solves 6 SAT and 
1 UNSAT instance uniquely, and \zspacer solves 3 unique SAT problems. 
\golemNested significantly outperforms all competitors for the UNSAT systems.
In analysis of SAT instances \golemNested is competitive with other techniques, it
is only slightly outperformed by \eldarica, which solves 6 more SAT problems.
To address \textbf{RQ2}, our evaluation shows that \golemNested not only solves the highest number of 
benchmarks but also does so more efficiently. 
On average, for instances solved by both solvers, 
\golemNested is 2.5$\times$ faster than \eldarica and 1.9$\times$ faster than \zspacer.
In summary, the proposed algorithm achieves strong performance on both SAT and UNSAT CHC instances, 
outperforming existing \golem engines and other state-of-the-art solvers.
