\section{Termination Analysis with Interpolation-based Transition Invariant Generation}

In this section, we present our novel approach for termination analysis of transition systems, which leverages interpolation-based transition invariant generation, building on the safety-based nontermination analysis.
The core idea of this approach is to use the terminating traces, detected by the safety-based nontermination analysis, to generate well-founded transition invariant.
The key contribution is a post-processing step triggered upon detection of a terminating trace.
The algorithm then computes an over-approximation of the transition trace, which it transforms into a disjunctively well-founded relation.
If this formula is a transition invariant over the reachable states, then the transition system is terminating.
Otherwise, the algorithm identifies states not covered by the transition invariant and recursively analyses them for termination.



\begin{algorithm}[t!]
	\SetInd{0.5em}{0.5em}
    \DontPrintSemicolon
    \small
    \newcommand{\Continue}{\textbf{continue}}
    \caption{Analyze(P)}
    \label{Alg:SNA_Term}
    \SetKwInOut{Input}{Input}
    \SetKwInOut{Output}{Output}
    \Input{Safety problem $P = \tuple{Init, Tr, Sink, x}$}
    \Output{$\tuple{NONTERM, R(x)}$ | $\tuple{TERM, TInv(x,x')}$}
    \While{$\top$} {
        $TInv \gets \bot$ \\
        \Switch{$safetyCheck(\tuple{Init, Tr, Sink, x})$}{
            \Case{$\tuple{UNSAFE, trace(x^{(0)}, \dots, x^{(n)})}$} {
                -- SAME AS IN ALGORITHM 2 -- \\
                $T^n(x), \forall x. T^n(x) \rightarrow \forall x^{(1)}, \dots, x^{(n)}. trace(x, x^{(1)}, \dots, x^{(n)}) \rightarrow Sink(x^{(n)})$  \\
                $Cs(x,x') \gets DNFIZE(Itp(trace(x, x^{(1)}, \dots, x'), T^n(x) \land \lnot Sink(x')))$ \\
                $TInv(x,x') \gets TInv(x,x') \lor \bigvee_i d_i$\tcp*[r]{$\bigvee_i d_i$ are w-f disjuncts $\in Cs$} 
                $Nc(x) \gets \exists x',x''. Tr(x,x') \land (Id(x',x'') \lor TInv(x',x'')) \land \lnot TInv(x,x'')$ \\
                \Switch{$safetyCheck(\tuple{Init, Tr, Nc, x})$}{ 
                    \label{SNA_Term:CheckNc}
                    \Case{$\tuple{SAFE, Inv(x)}$} {
                         \Return $\tuple{TERM, TInv(x,x')}$\label{SNA_Term:NEWTERM}
                    }
                    \Case{$\tuple{UNSAFE, trace(x^{(0)}, \dots, x^{(m)})}$} {
                        $M \models Init(x^{(0)}) \land trace(x^{(0)}, \dots, x^{(m)}) \land Sink(x^{(m)})$ \label{SNA_Term:UNSAFE}  \\
                        $s_m(x) \gets $ \textit{variable assigment extracted from } $M$ for $x^{(m)}$\\
                        \Switch{$Analyze(\tuple{s_m(x), Tr, Sink, x})$} {
                            \Case{$\tuple{NONTERM, R(x)}$} {
                                \Return $\tuple{NONTERM, R(x)}$ \label{SNA_Term:NEWNONTERM}
                            }
                            \Case{$\tuple{TERM, TInv'(x,x')}$} {
                                $TInv(x,x') \gets TInv(x,x') \lor TInv'(x,x')$ \\
                            }
                        }
                    }       
                
                }
            }
            \Case{$\tuple{SAFE, Inv(x)}$} {
                 \label{SNA_Term:SAFE}
                \lIf {UNSAT ? $Init(x) \land Tr(x,x')$} {
                    \Return $\tuple{TERM, TInv(x,x')}$\label{SNA_Term:OLDTERM}
                }
                \If{$\forall x. Inv(x) \rightarrow \exists x'. Tr(x,x') \land Inv(x')$} {
                    \Return $\tuple{NONTERM, Inv(x)}$ \label{SNA_Term:OLDNONTERM}
                } 
                $Sink(x) \gets \forall x. Sink'(x) \rightarrow \lnot \exists x'. Tr(x,x')$
            }
        }
    }
    
\end{algorithm}

Algorithm uses the terminating trace to generate states $T^n$, that necessarily lead to termination in up to $n$ steps of transition relation.
Then, using interpolation, the algorithm construct an over-approximation of the $n \leq i \leq 1$ steps of transition relation.
This interpolant is then transformed into a Disjunctive Normal Form (DNF), and every disjunct is separately checked for well-foundness, using linear Ranking function synthesis~\cite{DBLP:conf/lics/PodelskiR04}.
The disjunction of all well-founded disjuncts is then added to the candidate transition invariant formula $TInv$.

\begin{lemma}
    If Algorithm~\ref{Alg:SNA_Term} returns $\tuple{TERM, TInv(x,x')}$, then the transition system $\tuple{Init, Tr, x}$ is terminating.
\end{lemma}
\begin{proof}
    Algorithm~\ref{Alg:SNA_Term} returns $\tuple{TERM, TInv(x,x')}$ only at the line~\ref{SNA_Term:NEWTERM} or line~\ref{SNA_Term:OLDTERM}.
    In the first case, the algorithm returns $\tuple{TERM, TInv(x,x')}$.
    By construction, $TInv(x,x')$ is a disjunction of well-founded relations, each synthesised via linear ranking functions, so $TInv$ is disjunctively well-founded.

    It remains to show that $TInv$ is a transition invariant over all reachable states. 
    The safety check at line~\ref{SNA_Term:CheckNc} returning SAFE implies that $Nc(x)$ is unreachable from $Init(x)$ under $Tr'$. 
    Additionally, by construction, $Nc(x)$ captures all states for which $TInv$ is not a transition invariant.
    It means that for states satisfying $\lnot Nc(x)$ $TInv$ is a transition invariant under $Tr'$:
    $$\forall x,x',x''. \lnot Nc(x) \land (TInv(x,x') \lor Id(x,x')) \land Tr'(x',x'') \land \lnot Nc(x'') \rightarrow TInv(x,x'')$$

    In the formula $Tr'(x,x') = Tr(x,x') \land \lnot \bigvee_i s_i(x')$, where $\bigvee_i s_i(x')$ are the states that have 
    been blocked during the analysis and are guaranteed to terminate.
    $TInv$ is and therefore is a disjunctively well-founded transition invariant over the states described by $\lnot Nc(x) \land \lnot \bigvee_i s_i(x')$.

    Since (1) $Nc(x)$ is not reachable from $Init(x)$ based on the check at line~\ref{SNA_Term:CheckNc}, (2) states in $\bigvee_i s_i(x')$ are guaranteed to terminate, and (3) $TInv$ is a transition invariant over all remaining states
     - the transition system $\tuple{Init, Tr, x}$ is terminating~\cite{DBLP:conf/lics/PodelskiR04}.

    In the second case, the proof follows directly from Lemma~\ref{Lem:TrivialTermination}, as it is based on the same check of $Init(x) \land Tr(x,x')$.
\end{proof}


\begin{lemma}
    If Algorithm~\ref{Alg:SNA_Term} returns $\tuple{NONTERM, R(x)}$, then the transition system $\tuple{Init, Tr, x}$ is nonterminating.
\end{lemma}
\begin{proof}
    Algorithm~\ref{Alg:SNA_Term} returns $\tuple{NONTERM, R(x)}$ at the line~\ref{SNA_Term:NEWNONTERM} or line~\ref{SNA_Term:OLDNONTERM}.
    At line~\ref{SNA_Term:OLDNONTERM}, the algorithm returns $\tuple{NONTERM, Inv(x)}$, where $Inv(x)$ is a recurrent set, which implies that the transition system is nonterminating (following from the Lemma~\ref{Lem:NontermPreservation}).
    
   Another case is when the algorithm returns the result of a recursive call $Analyze(\tuple{s_m(x), Tr, Sink, x})$. 
   By the inductive hypothesis applied to this call, $R(x)$ is a recurrent set for $\tuple{s_m, Tr, x}$, satisfying:
    \begin{enumerate}[label=\Roman*.]
        \item $\exists x.\ s_m(x) \land R(x)$, and
        \item $\forall x.\ R(x) \rightarrow \exists x'. Tr'(x,x') \land R(x')$.
    \end{enumerate}
    Since $s_m(x)$ is reachable from $Init(x)$ (line~\ref{SNA_Term:CheckNc}), $R(x)$ is also reachable from $Init(x)$, and therefore $R(x)$ is a recurrent set for the original transition system, which implies that the transition system is nonterminating.
\end{proof}

\section{Safety-based Nontermination Analysis Extensions}
\label{Sec:Main}

In this section, we present our Transition System-based extensions for safety-based termination analysis, which allow to prove termination of transition systems and to improve the nontermination analysis. 


\subsection{Extension of the Algorithm for Termination of Transition Systems}

Algorithm~\ref{Alg:SNA} can be extended to prove termination of transition systems via a single additional SMT check.
Specifically, when the algorithm reaches the SAFE case at line~\ref{SNA:SAFE}, the following check is performed:
$$Init(x) \land Tr(x,x')$$
This check is intended to determine whether it is possible to take a transition after all traces to sink states have been blocked.
Throughout the execution of the algorithm, the transition relation is progressively refined to exclude states that deterministically lead to sink states.
If this formula is unsatisfiable, then the transition system is terminating (since Init states lead to sink states).


\subsection{Safety Problem Refinement}

The algorithm can further refine the safety problem $P$ in the case where $Inv$ does not constitute a recurrent set.
Particularly, it is possible to extract states within the invariant, from which it is impossible to take a transition (which makes $Inv$ not a recurrent set).

$$\forall x. Sink'(x) \land \lnot \exists x'. Tr'(x,x')$$

In this formula, $Sink'(x)$ is a new formula which describes sink states within the inductive invariant.
It can be computed  by applying quantifier elimination to $\lnot \exists x'. Tr'(x,x')$.
Afterwards, the produced formula $Sink'(x)$ replaces $Sink(x)$ in the safety problem $P$, enabling further analysis.

This algorithm differs from the original one in two lines.
Particularly, at line~\ref{SNA_Refined:TERM} the algorithm checks if one can identify states within the invariant from which no transition is possible — precisely those states that prevent $Inv$ from being a recurrent set.
It allows to check the termination in some cases.
And at line~\ref{SNA_Refined:SAFE} the algorithm refines the sink states, by extracting states from the invariant, from which it is impossible to take a transition.
Which allows to proceed with termination analysis and allows to solve more complex problems.


\begin{lemma}\label{Lem:TrivialTermination}
    If the formula $Init(x) \land Tr(x,x')$ is unsatisfiable at line~\ref{SNA_Refined:TERM} of Algorithm~\ref{Alg:SNA_Refined}, then the transition system $\tuple{Init, Tr, x}$ is terminating.
\end{lemma}
\begin{proof}
    The algorithm maintains the following invariants throughout its execution:
    \begin{align}
        Tr'(x,x')   &= Tr(x,x') \land \textstyle\bigwedge_i \lnot s_i(x') \label{eq:TrRefined}\\
        Init'(x)    &= Init(x)  \land \textstyle\bigwedge_j \lnot s_j(x) \label{eq:InitRefined}
    \end{align}
    where each blocked state $s_i(x')$ (resp. $s_j(x)$) is one that deterministically leads to a sink state, and is therefore guaranteed to terminate if reached.

    Suppose $Init'(x) \land Tr'(x,x')$ is unsatisfiable.  
    Expanding the definitions gives:
    $$
      \bigl(Init(x) \land \textstyle\bigwedge_j \lnot s_j(x)\bigr) 
      \;\land\; 
      \bigl(Tr(x,x') \land \textstyle\bigwedge_i \lnot s_i(x')\bigr)
      \;\equiv\; \bot
    $$
    This means that every initial state from which a transition is still possible has been blocked, i.e., every reachable initial state either cannot take a transition, or can only reach a state in $\{s_i\}$ or originate from a state in $\{s_j\}$, all of which are guaranteed to terminate. 
    Since no infinite trace can be initiated from any such state, the transition system is terminating.
\end{proof}

\begin{lemma}\label{Lem:NontermPreservation}
    If the refined transition system $\tuple{Init', Tr', x}$ is nonterminating, then the original transition system $\tuple{Init, Tr, x}$ is also nonterminating.
\end{lemma}
\begin{proof}
    Suppose $\tuple{Init', Tr', x}$ is nonterminating. 
    Then there exists a recurrent set $Inv'(x)$ such that:
    \begin{enumerate}[label=\Roman*.]
        \item $\exists x.\ Init'(x) \land Inv'(x)$, and
        \item $\forall x.\ Inv'(x) \rightarrow \exists x'.\ 
              Tr'(x,x') \land Inv'(x')$.
    \end{enumerate}
    We show that $Inv'(x)$ is also a recurrent set for the original system $\tuple{Init, Tr, x}$.

    \textbf{Reachability from $Init$.} Since $Init'(x) = Init(x) \land \bigwedge_i \lnot s_i(x)$, we have $Init'(x) \rightarrow Init(x)$. 
    Combined with condition~(I), there exists a state $x_0$ such that $Init(x_0) \land Inv'(x_0)$ holds, so $Inv'$ is reachable from $Init$.

    \textbf{Closure under $Tr$.} Since $Tr'(x,x') = Tr(x,x') \land \bigwedge_i \lnot s_i(x')$, we have $Tr'(x,x') \rightarrow Tr(x,x')$. 
    Therefore, for any $x$ with $Inv'(x)$, condition~(II) guarantees the existence of $x'$ with $Tr'(x,x') \land Inv'(x')$, and hence $Tr(x,x') \land Inv'(x')$.

    Thus $Inv'(x)$ satisfies both conditions of a recurrent set for $\tuple{Init, Tr, x}$, which is therefore nonterminating.
\end{proof}
