\section{Termination Analysis via Interpolation-based Transition Invariant Generation}

We propose a new procedure for proving termination that leverages safety verification to generate a well-founded transition invariant via interpolation.
At each iteration, our procedure constructs a trace from the initial states to a sink state.
A well-founded transition invariant is then constructed based on the overapproximation of the terminating trace.
This transition invariant proves termination if it is valid for all reachable states.
Otherwise, our algorithm uses transition invariant for the refinement of the state space, focusing on states 
which are not covered by such an invariant.
The interpolation-based approach enables efficient property-directed transition invariant generation, as it is constructed with respect to the termination condition.

\subsection{Interpolation-based Generation of Well-Founded Transition Invariants}

The core of our approach is the automated, interpolation-based generation of 
well-founded transition invariants. Given a terminating trace, the procedure 
constructs an over-approximation of the transition relation, guided by the sink 
states, and filters it into a disjunction of well-founded relations. If the 
resulting formula constitutes a transition invariant over all reachable states, 
it witnesses termination of the transition system.

The Procedure consists of five major steps:
(1) computing $T^n$, the formula encoding states that terminate within $n$ transitions; 
(2) constructing an over-approximation of the terminating trace via interpolation; 
(3) converting the interpolant into a underapproximating DNF formula; 
(4) filtering disjuncts to retain only those encoding well-founded relations; and 
(5) computing the formula encoding states for which the produced formula is a valid transition invariant.
The overall flow of the procedure can be seen in the figure~\ref{fig:TrInvGen}.

Algorithm~\ref{Alg:SNA_TrInv} takes as an input a safety problem $P = \tuple{Init, Tr, Sink, x}$, a 
transition trace $Tr^n(x^{(0)}, x^{(1)}, \dots, x^{(n)})$, and a transition invariant formula $TInv(x,x')$. 
It returns a tuple of formulas $\tuple{TInv, C}$, where $TInv(x,x')$ is a transition invariant formula, and $C(x)$ is
a formula encoding the states for which $TInv$ is a valid transition invariant. 
It proceeds in the following steps:

\paragraph{Step 1} \textit{Computing $T^n$ (line~\ref{SNA_Term:T}).} 
Formula $T^n$ is defined by the following property:
$$\forall x^{(0)}, \dots, x^{(n)}. T^{n}(x^{(0)}) \land Tr^{n}(x^{(0)}, \dots, x^{(n)}) \rightarrow Sink(x^{(n)})$$

$T^n$ is constructed with quantifier elimination, eliminating all variables from the formula except for $x^{(0)}$, $T^{n}(x^{(0)}) =$
% Give an example of what is T^n +
% Give definitions
$$\lnot QE(\exists x^{(1)}, \dots, x^{(n)}. Tr^{n}(x, x^{(1)}, \dots, x^{(n)}) \land \lnot Sink(x^{(n)}))$$

Construction of such states enables the interpolation over the transition relation. 
\begin{example}\label{Ex:Tn}
    Consider a terminating transition system $TS = \tuple{Init, Tr, x}$ in which:
    \begin{itemize}
        \item $Init \triangleq  v > 0$
        \item $Tr(x,x') \triangleq  ((v' = v - 1 \lor v' = v - 2) \land v > 0) \lor (v < -10 \land v' = v - 1)$
        \item $x \triangleq  \{v\}$
    \end{itemize}
    For example, for $n = 2$, $T^2(x) = v \leq 2$.
    Even though $TS$ can reach termination in 2 transitions with $v=3 \lor v=4$, those states are not included in $T^n$, since they do not necessarily terminate within 2 transitions.
    $v=1 \lor v=2$ on the other hand do terminate in 1 or 2 transitions.
\end{example}

\paragraph{Step 2} \textit{Interpolation (line ~\ref{SNA_Term:Itp}).}
We construct an overapproximation of the trace via Craig interpolation. The following formula is unsatisfiable by construction:

$$T^{n}(x^{(0)}) \land Tr^{n}(x^{(0)}, \dots, x^{(n)}) \land \lnot Sink(x^{(n)})$$

This allows to compute the \textbf{interpolant}, which over-approximates the trace $Tr^{n}$.
The key intuition is that the usage of the sink states for interpolant construction allows the interpolant to capture the reason \textbf{why} 
the particular trace is leading to the termination.


\paragraph{Step 3} \textit{DNF underapproximation (line ~\ref{SNA_Term:toDNF}).}
For transition invariant to be a proof of termination, it needs to be disjunctively well-founded, meaning every disjunct must 
correspond to a well-founded relation.
Full conversion to a DNF format is expensive and can cause an exponential blowup in formula size. 
Instead of full conversion, our algorithm uses a
heuristic approach to construct a DNF formula, which underapproximates the interpolant.
It constructs disjuncts from the interpolant using the syntactic analysis, bounded by a constant to avoid exponential blowup.
It does not violate the correctness of the approach, as in the worst case the resulting formula would not be a transition invariant.

\paragraph{Step 4} \textit{Filtering Well-Founded Disjuncts (line ~\ref{SNA_Term:TINV}).}
For each disjunct $d(x,x') \in Cands(x,x')$ algorithm checks if $d(x,x')$ corresponds to a well-founded relation.
Our technique uses automated linear ranking function synthesis~\cite{DBLP:conf/vmcai/PodelskiR04} for checking the existence of a ranking function.
The disjuncts for which a ranking function exists are preserved, and the rest are discarded.
The resulting formula is disjoined with the input candidate transition invariant $TInv$.

\paragraph{Step 5} \textit{Synthesis of the states for which transition invariant is valid (line ~\ref{SNA_Term:NC}).}
If $TrInv$ is a transition invariant over all reachable states, then TS terminates, since every disjunct corresponds to well-founded relation.
To verify this, procedure computes the formula $C(x)$ encoding the states over which $TrInv$ is a valid transition invariant. 
The formula $C(x)$ is defined by the following property:

\begin{align*}
    \forall x, x',x''. C(x) \land (TrInv(x,x') \lor& Id(x,x')) \land Tr(x',x'') \\ 
    \rightarrow& TrInv(x,x'')
\end{align*}

Similarly to $T^{n}$, $C(x)$ is constructed using the quantifier elimination, negating the states for 
which $TrInv$ is not a transition invariant.

\begin{example}~\label{Ex:C}
    Consider the transition system from the Example~\ref{Ex:Tn}.
    Let's assume that the procedure was able to construct a transition invariant $TrInv(x,x') \triangleq v' < v \land v' \geq 0$.
    Then the quantifier elimination would produce $C(x) \triangleq v \geq 0$, as transition invariant is defined only for 
    the states that satisfy $C(x)$.
\end{example}

\emph{Note.} If $\lnot C(x)$ is not reachable, then $TrInv(x,x')$ is a disjunctively well-founded
transition invariant over all reachable states, and it witnesses the termination of transition system.
For illustration, transition invariant in the Example~\ref{Ex:C} is sufficient to prove termination, since 
$\lnot C(x) \triangleq v < 0$ is not reachable in the transition system.

\begin{lemma}\label{Lem:Term}
    Let $TS = \tuple{Init, Tr, x}$ be a transition system and let 
    $C(x)$ be a formula such that $\lnot C(x)$ is unreachable in 
    $TS$. If $TrInv(x,x')$ is a union of well-founded 
    relations and satisfies:
    \begin{align*}
        \forall x, x',x''. C(x) \land (TrInv(x,x') \lor& Id(x,x')) \land Tr(x',x'') \\ 
        \rightarrow& TrInv(x,x'')
    \end{align*}    
then $T$ is terminating.
\end{lemma}
\begin{proof}
    We show that $TrInv$ is a disjunctively well-founded transition 
    invariant over the reachable states of $T$, from which 
    termination follows by~\cite{DBLP:conf/lics/PodelskiR04}.

    \textbf{Well-foundedness.} By construction, for each disjunct 
    $d_i$ of $TrInv = \bigvee_i d_i$ there exists a ranking function.
    Therefore, every disjunct is a logical encoding of 
    well-founded relation~\cite{DBLP:conf/vmcai/PodelskiR04}. 
    Hence $TrInv$ is disjunctively well-founded.

    \textbf{Transition invariant.} We show that 
    $\forall x,x'.\ R(x) \land Tr^*(x,x') \rightarrow TrInv(x,x')$,
    where $R(x)$ denotes the reachable states of $T$, by induction 
    on the number of steps $n$ in $Tr^*(x,x')$.

   \emph{Base case} ($n = 1$): $Tr^1(x,x') \equiv Tr(x,x')$. 
    Since $\lnot C(x)$ is unreachable, $R(x) \rightarrow C(x)$. 

    The following assumption holds originally by the lemma statement:
    $$C(x) \land Id(x,x') \land Tr(x',x'') \rightarrow TrInv(x,x'')$$
    Since $Id(x,x')$ is an identity relation, it can be simplified to:
    $$C(x) \land Tr(x,x') \rightarrow TrInv(x,x').$$
    Since $R(x) \rightarrow C(x)$, the base case holds.

   \emph{Inductive step}: Assume $R(x) \land Tr^n(x, \dots,x') \rightarrow 
    TrInv(x,x')$ for some $n > 0$. 
    Let $Tr^{n+1}(x,x'')$ hold, so there exists $x, x', x''$ with $R(x) \land Tr^n(x, \dots, x') \land Tr(x',x'')$. 
    By the inductive hypothesis, $TrInv(x,x')$ holds. 
    Since $R(x) \land Tr^n(x, \dots, x') \rightarrow TrInv(x,x')$, then trivially
    $R(x) \land Tr^n(x,x') \land Tr(x',x'') \rightarrow R(x) \land TrInv(x,x') \land Tr(x',x'')$.
    Now, due to the fact that $R(x) \rightarrow C(x)$, the following holds:
    $$R(x) \land TrInv(x,x') \land Tr(x',x'') \rightarrow C(x) \land TrInv(x,x') \land Tr(x',x'').$$
    And by claim in the lemma:
    $$C(x) \land TrInv(x,x') \land Tr(x',x'') \rightarrow TrInv(x,x'').$$

    Therefore, through the chain of implications, we have:
    $$R(x) \land Tr^{n+1}(x,x') \rightarrow TrInv(x,x').$$

    Thus $TrInv$ is a disjunctively well-founded transition 
    invariant over all reachable states, and $T$ is 
    terminating~\cite{DBLP:conf/lics/PodelskiR04}.
\end{proof}

\subsection{Interpolation-based Termination Analysis}


The algorithm description is given in Algorithm~\ref{Alg:SNA_Term}. 
Below, we explain the steps of the algorithm in detail.
Then, provide an algorithm execution example.
Afterwards, we prove the correctness of the introduced approach.

Our technique, similarly to the Algorithm~\ref{Alg:SNA} takes as input a Safety Problem $P=\tuple{Init, Tr, Sink, x}$, where $Init$ and 
$Tr$ encode the initial states and transition relation, $Sink$ is the formula encoding sink states, and $x$ is 
a set of variables over which the Transition System is defined.
As an output, the algorithm returns NONTERM if TS is nonterminating, TERM if it is terminating, and UNKNOWN if the procedure cannot determin the final result.

\begin{algorithm}[t!]
	\SetInd{0.5em}{0.5em}
    \DontPrintSemicolon
    \small
    \newcommand{\Continue}{\textbf{continue}}
    \caption{Interpolation-based Termination Analysis.}
    \label{Alg:SNA_Term}
    \SetKwInOut{Input}{Input}
    \SetKwInOut{Output}{Output}
    \Input{Safety problem $P = \tuple{Init, Tr, Sink, x}$}
    \Output{TERM | NONTERM | UNKNOWN}
    $TInv \gets \bot$ \\
    \While{$\top$} {
        \label{SNA_Term:Safety} $\tuple{res, \phi} \gets safetyCheck(P)$ \\
        \eIf{$\tuple{res, \phi} = \tuple{UNSAFE, Tr^{n}(x^{(0)}, \dots, x^{(n)})}$} {
            $\tuple{TInv, C} \gets TrInvSynthesis(P, Tr^n, TInv)$ \\
            \label{SNA_Term:CheckNc} $\tuple{res', \phi'} \gets safetyCheck(\tuple{Init', Tr', \lnot C, x})$ \\
            \eIf{$\tuple{SAFE, Inv(x)}$} {
               \Return $TERM$\label{SNA_Term:NEWTERM}
            }
            {
                $P \gets \blockTerm(P, Tr^{n})$ \\
            }
        }{
            \lIf{$\forall x. Inv(x) \rightarrow \exists x'. Tr(x,x') \land Inv(x')$} {
                \Return $NONTERM$ \label{SNA_TERM:NONTERM}
            }
            \Return UNKNOWN
        }
        
    }
\end{algorithm}


The procedure maintains two important variables: $TInv$ - formula that serves as a transition invariant candidate, it is  progressively extended
 by adding well-founded disjuncts. 
$Prs$ is a vector of safety problems, which contains the safety problems that need to be analized within the algorithm to complete
the termination proof.

Algorithm commences by analyzing the original safety problem.
It iterates until it either finds a recurrent set - proving nontermination, or finds a disjunctively well-founded Transition Invariant, 
that covers the reachable state space.
Every iteration, similarly to the Algorithm~\ref{Alg:SNA}, begins with the safety check at line~\ref{SNA_Term:Safety}, extracting the last 
element from the list of safety problem (which also decreases the size of $Prs$).
The algorithm handles two possible cases:

\paragraph{UNSAFE.} It means there exists a terminating trace in the Transition System.
First, algorithm executes a \blockTerm procedure, blocking the determenistically terminating states.
It adds a constructed problem into the list of safety problems.
At the lines~\ref{SNA_Term:T}-\ref{SNA_Term:CheckNc} algorithm executes the interpolation-based generation of transition invariant 
candidate, following the approach described above - generating $T^n$, constructing an overappproximation of transition trace and
transforming it into union of well-founded relations.


If it is possible to reach state in $\lnot C(x)$ from the initial states, then our approach tries to prove termination of this state, not covered by 
transition invariant, by updating $Init'(x)$ to this state.
If $C(x)$ is not reachable from the updated Initial states $Init'(x) \neq Init(x)$, then algorithm checks, if $Prs$ is empty,
meaning that the termination of original transition system was proven, and if it is the case - it concludes that TS is terminating. 
Otherwise, if $C(x)$ is reachable from the original Initial states $Init(x)$ - algorithms constructs the formula, encoding some 
states noncovered by the transition invariant, reachable from the initial states - $s_m$. 
Afterwards is adds a safety problem $\tuple{x = s_m, Tr', Sink', x}$ to $Prs$, attempting
to either extend transition invariant, proving the termination of $s_m$, or to find a recurrent set.

\paragraph{SAFE.} In case $Sink$ states are not reachable, and $Inv(x)$ is a recurrent set - TS is nonterminating.
Otherwise, $NONDET$ as the procedure cannot conclude the result.

\begin{example}
    Consider a terminating transition system $T$ in which:
    \begin{itemize}
        \item $Init: \top$
        \item $Tr(x,x'): (v' = v - 1 \lor v' = v - 2) \land v > 0$
        \item $x: \{v\}$
    \end{itemize}

    Initially, before the algorithm execution, preprocessing constructs the sink states: 
    $$Sink(x) = QE(\lnot Tr(x,x'),\tuple{x'}) = v \leq 0.$$
    For any state with $v \leq 0$ it is impossible to do a transition in the TS.
    Additonally, algorithm would update initial states: 
    $$Init'(x) = Init(x) \land \lnot Sink(x) = v > 0.$$
    Since every state that satisfies sink states will terminate by default.

    %

    The algorithm is called with $P=\tuple{Init', Tr, Sink, x}$.
    First, it executes the safety check, detecting the trace that leads to the sink states.
    For example, it detects a trace of length 2:  $ Init(x) \land Tr^2(x,x',x'') \land Sink(x)$.
    Then, it first constructs $T^2(x)$, formula for all of the states that are guaranteed to terminate in one or less transitions:
    $$T^2(x) = v \leq 2$$
    Then, it constructs an interpolant:
    $$Itp(Tr^{2}(x,x^{(1)},x'), T^2(x) \land \lnot Sink(x')) = v' \leq v - 1 \land v' > 0.$$

    This interpolant is a conjunction, therefore the dnfization returns the same formula in the line~\ref{SNA_Term:Itp}.
    Using the linear ranking function synthesis method, it is proven that $v' \leq v - 1 \land v > 0$ is well-founded, and the 
    transition invariant candidate $TInv(x,x') = v' \leq v - 1 \land v > 0$.
    At the line~\ref{SNA_Term:NC}, algorithm constructs states that are not covered by the invariant:
    $$Nc(x) = v \leq 1$$
    The safety check then detects that $Nc(x)$ is not reachable from the initial states, therefore transition invariant candidate
    $TInv$ covers all the reachable states.
    Since $TInv$ is well-founded, algorithm concludes that $T$ terminates.
\end{example}


\subsection{Correctness}

This subsection proves the soundness of the introduced approach. We first prove that when algorithm returns TERM, the transition system is nonterminating,
and then we prove the correctnessof the nontermination.

\begin{lemma}
    If Algorithm~\ref{Alg:SNA_Term} returns $TERM$, then the transition system $\tuple{Init, Tr, x}$ is terminating.
\end{lemma}
\begin{proof}
    Algorithm~\ref{Alg:SNA_Term} returns $TERM$ at the line~\ref{SNA_Term:NEWTERM}.
    If algorithm returns $TERM$ then the safety check at line~\ref{SNA_Term:CheckNc} returns SAFE.
    The termination of the transition system $T = \tuple{Init', Tr', x}$ follows from the lemma~\ref{Lem:Term}.
    Based on the construction (line~\ref{SNA_Term:Extension}), the last system is:
    $Init'(x) = Init(x) \land \bigwedge_{i=0}^{m} c_m(x)$, where every $c_m(x)$ encodes states in initial states formula,
    which determenistically terminate (due to the \blockTerm).


    It implies that $Nc(x)$ is unreachable from $Init(x)$ under $Tr'$. 
    Additionally, by construction, $Nc(x)$ captures all states for which $TInv$ is not a transition invariant.
    It means that for states satisfying $\lnot Nc(x)$ $TInv$ is a transition invariant under $Tr'$:
    $$\forall x,x',x''. \lnot Nc(x) \land (TInv(x,x') \lor Id(x,x')) \land Tr'(x',x'') \land \lnot Nc(x'') \rightarrow TInv(x,x'')$$
    By construction, $TInv$ is also disjunctively well-founded (line~\ref{SNA_Term:TINV}).

    Since (1) $Nc(x)$ is not reachable from $Init(x)$ based on the check at line~\ref{SNA_Term:CheckNc}, and (2) $TInv$ is a disjunctively well-founded transition invariant over all remaining states
     - the transition system $\tuple{Init, Tr, x}$ is terminating~\cite{DBLP:conf/lics/PodelskiR04}.
\end{proof}


\begin{lemma}
    If Algorithm~\ref{Alg:SNA_Term} returns $NONTERM$, then the transition system $\tuple{Init, Tr, x}$ is nonterminating.
\end{lemma}
\begin{proof}
    The proof of this lemma follows directly from the lemma~\ref{Lem:NONTERM}. 
    To prove it we prove the existence of the recurrent set, for which two conditions should hold:
    \begin{itemize}
        \item $\forall x. N(x) \rightarrow \exists x'. Tr(x,x') \land N(x')$
        \item $\exists x. N(x) \land Init(x)$
    \end{itemize}
    \emph{First condition:} For $Inv(x)$ the following holds: $\forall x. Inv(x) \rightarrow \exists x'. Tr(x,x') \land Inv(x')$.
    This is the case as two properties below hold:
    \begin{itemize}
        \item $\lnot Sink(x) \rightarrow \exists x'. Tr(x,x')$ (by construction of $Sink(x)$)
        \item $Inv(x) \rightarrow \lnot Sink(x)$ (As a result of safety check)
    \end{itemize}

    \emph{Second condition.} For any safety problem $\tuple{Init', Tr, Sink}$, $Init'$ is reachable from the initial states $Init$ by construction.
    The states that are reached on the trace from $Init$ to $Init'$ can be added to $Inv(x)$ to construct a full recurrent set.

    Therefore, there exists a recurrent set within the transition system, and it is nonterminating.
\end{proof}

\section{Extension of the Algorithm for Termination of Transition Systems}

Algorithm~\ref{Alg:SNA} can be extended to prove termination of transition systems via a single additional SMT check.
Specifically, when the algorithm reaches the SAFE case at line~\ref{SNA:SAFE}, the following check is performed:
$$Init(x) \land Tr(x,x')$$
This check is intended to determine whether it is possible to take a transition after all traces to sink states have been blocked.
Throughout the execution of the algorithm, the transition relation is progressively refined to exclude states that deterministically lead to sink states.
If this formula is unsatisfiable, then the transition system is terminating (since Init states lead to sink states).


\subsection{Safety Problem Refinement}

The algorithm can further refine the safety problem $P$ in the case where $Inv$ does not constitute a recurrent set.
Particularly, it is possible to extract states within the invariant, from which it is impossible to take a transition (which makes $Inv$ not a recurrent set).

$$\forall x. Sink'(x) \land \lnot \exists x'. Tr'(x,x')$$

In this formula, $Sink'(x)$ is a new formula which describes sink states within the inductive invariant.
It can be computed  by applying quantifier elimination to $\lnot \exists x'. Tr'(x,x')$.
Afterwards, the produced formula $Sink'(x)$ replaces $Sink(x)$ in the safety problem $P$, enabling further analysis.

This algorithm differs from the original one in two lines.
Particularly, at line~\ref{SNA_Refined:TERM} the algorithm checks if one can identify states within the invariant from which no transition is possible — precisely those states that prevent $Inv$ from being a recurrent set.
It allows to check the termination in some cases.
And at line~\ref{SNA_Refined:SAFE} the algorithm refines the sink states, by extracting states from the invariant, from which it is impossible to take a transition.
Which allows to proceed with termination analysis and allows to solve more complex problems.


\begin{lemma}\label{Lem:TrivialTermination}
    If the formula $Init(x) \land Tr(x,x')$ is unsatisfiable at line~\ref{SNA_Refined:TERM} of Algorithm~\ref{Alg:SNA_Refined}, then the transition system $\tuple{Init, Tr, x}$ is terminating.
\end{lemma}
\begin{proof}
    The algorithm maintains the following invariants throughout its execution:
    \begin{align}
        Tr'(x,x')   &= Tr(x,x') \land \textstyle\bigwedge_i \lnot s_i(x') \label{eq:TrRefined}\\
        Init'(x)    &= Init(x)  \land \textstyle\bigwedge_j \lnot s_j(x) \label{eq:InitRefined}
    \end{align}
    where each blocked state $s_i(x')$ (resp. $s_j(x)$) is one that deterministically leads to a sink state, and is therefore guaranteed to terminate if reached.

    Suppose $Init'(x) \land Tr'(x,x')$ is unsatisfiable.  
    Expanding the definitions gives:
    $$
      \bigl(Init(x) \land \textstyle\bigwedge_j \lnot s_j(x)\bigr) 
      \;\land\; 
      \bigl(Tr(x,x') \land \textstyle\bigwedge_i \lnot s_i(x')\bigr)
      \;\equiv\; \bot
    $$
    This means that every initial state from which a transition is still possible has been blocked, i.e., every reachable initial state either cannot take a transition, or can only reach a state in $\{s_i\}$ or originate from a state in $\{s_j\}$, all of which are guaranteed to terminate. 
    Since no infinite trace can be initiated from any such state, the transition system is terminating.
\end{proof}

\begin{lemma}\label{Lem:NontermPreservation}
    If the refined transition system $\tuple{Init', Tr', x}$ is nonterminating, then the original transition system $\tuple{Init, Tr, x}$ is also nonterminating.
\end{lemma}
\begin{proof}
    Suppose $\tuple{Init', Tr', x}$ is nonterminating. 
    Then there exists a recurrent set $Inv'(x)$ such that:
    \begin{enumerate}[label=\Roman*.]
        \item $\exists x.\ Init'(x) \land Inv'(x)$, and
        \item $\forall x.\ Inv'(x) \rightarrow \exists x'.\ 
              Tr'(x,x') \land Inv'(x')$.
    \end{enumerate}
    We show that $Inv'(x)$ is also a recurrent set for the original system $\tuple{Init, Tr, x}$.

    \textbf{Reachability from $Init$.} Since $Init'(x) = Init(x) \land \bigwedge_i \lnot s_i(x)$, we have $Init'(x) \rightarrow Init(x)$. 
    Combined with condition~(I), there exists a state $x_0$ such that $Init(x_0) \land Inv'(x_0)$ holds, so $Inv'$ is reachable from $Init$.

    \textbf{Closure under $Tr$.} Since $Tr'(x,x') = Tr(x,x') \land \bigwedge_i \lnot s_i(x')$, we have $Tr'(x,x') \rightarrow Tr(x,x')$. 
    Therefore, for any $x$ with $Inv'(x)$, condition~(II) guarantees the existence of $x'$ with $Tr'(x,x') \land Inv'(x')$, and hence $Tr(x,x') \land Inv'(x')$.

    Thus $Inv'(x)$ satisfies both conditions of a recurrent set for $\tuple{Init, Tr, x}$, which is therefore nonterminating.
\end{proof}
