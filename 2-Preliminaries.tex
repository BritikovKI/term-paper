\section{Preliminaries}
\label{Sec:Preliminaries}
This section introduces the definitions and formalisms used for the construction of the Termination Analysis techniques. 
Logical formulas in our approach are defined over the theory of Linear Integer Arithmetic (LIA).
Conventional primed notation is used to represent ``next-state'' variables.
In cases where there are multiple states, we use superscripts to distinguish them, e.g., $x^{(0)}, x^{(1)}, \dots$.
We use $s(x), s_1(x), s_2(x) \dots$ to denote the specific states of the system, which is a conjunction in which each variable in $x$ is assigned a specific value.
%
\paragraph{Transition systems and Safety Problem.} 
A \emph{transition system} is defined as $T = \tuple{ Init, Tr, x }$, where $Init$ is an initial state of the system represented by a first-order logic formula, $Tr$ is a transition formula, which represents the transition in the system, and $x$ is a set of system's variables. 
A \emph{safety problem} is defined as  $P = \tuple{ Init, Tr, Bad, x }$ where $Bad$ is a formula that represents a state violating the safety property. 
Safety verification amounts to determining whether there exists a transition trace through the system that reaches a BadBad
Bad state.
Safety problem $P$ is unsafe if there exists a satisfying assignment for the following formula:
$$Init(x^{(0)}) \land  Tr(x^{(0)}, x^{(1)}) \land \ldots \land Tr(x^{(n-1)}, x^{(n)}) \land Bad(x^{(n)})$$

If $P$ is SAFE, then there exists an inductive invariant $Inv(x)$ such that $Init(x) \implies Inv(x)$, $Inv(x) \land Tr(x,x') \implies Inv(x')$, and $Inv(x) \implies \lnot Bad(x)$.
We use $safetyCheck(P)$ to denote a safety verification procedure which returns a tuple $\tuple{res, \phi}$ where $res = SAFE$ if TS is safe and $\phi = Inv(x)$ (inductive state invariant), otherwise if TS is unsafe $res = UNSAFE$ and $\phi$ is a trace that leads to the bad states: 
$$\phi = trace(x^{(0)}, \dots, x^{(n)}) =  Tr(x^{(0)},x^{(1)}) \land \ldots \land Tr(x^{(n-1)},x^{(n)}).$$

\paragraph{Nontermination.}
A transition system is called \textbf{nonterminating} if there exists a set of states, called a \emph{recurrent set}, within which there always exists SAT transition to another state in recurrent set.
\begin{definition}[Recurrent Set]
    For a Transition System $\tuple{ Init, Tr, x }$, a recurrent set is a formula $N(x)$ such that:
    \begin{itemize}
        \item $\forall x. N(x) \implies \exists x'. Tr(x,x') \land N(x')$
        \item $\exists x. N(x) \land Init(x)$
    \end{itemize}
\end{definition}
Intuitively, every state in the recurrent set has a successor that also lies in the recurrent set, and the set is reachable from some initial state.
\paragraph{Termination.} A Transition System is called \textbf{Terminating} if for every initial state there does not exist an infinite satisfiable transition trace.
Termination can be proven by synthesizing a \emph{ranking function}~\cite{DBLP:conf/vmcai/PodelskiR04}
, which maps states to a well-founded domain such that its value strictly decreases along every transition.
Additionally, a \emph{transition invariant}~\cite{DBLP:conf/lics/PodelskiR04} 
can be used to prove termination.

\begin{definition}[Transition invariant]
    For a Transition System $\tuple{ Init, Tr, x }$, $TrInv(x,x')$ is a transition invariant if $Tr^*(x,x') \subset TrInv(x,x')$ over the set of reachable states.
    In a First Order Logic: $\forall x,x'. R(x) \land Tr^*(x,x') \land R(x') \implies TrInv(x,x')$, where $Tr^*(x,x')$ is the reflexive transitive closure of $Tr(x,x')$
    and $R(x)$ is a formula which describes all of the states, reachable within transition system.
\end{definition}

\emph{Transition invariant} proves the termination of transition system, if it can be expressed as a finite disjunction of well-founded relations.

\begin{definition}[Disjunctively well-founded transition invariant]
    A transition invariant $TrInv(x,x') = R_1(x,x') \lor R_2(x,x') \lor \dots \lor R_n(x,x')$ is called disjunctively well-founded if every disjunct $R_i(x,x')$ is well-founded. 
\end{definition}

\paragraph{Craig Interpolation.} This is a technique for computing an overapproximation of the formula~\cite{Craig_1957}.
Particularly, given two formulas $A$ and $B$ such that $A \land B$ is unsatisfiable, a Craig interpolant for $A$ and $B$ is a formula $I$ such that $A \implies I$, $I \land B$ is unsatisfiable, and $I$ only contains the symbols that are common to both $A$ and $B$.
We use $Itp(A, B)$ to denote the Craig interpolant for $A$ and $B$.

\paragraph{Proving Nontermination via Safety.}
Several techniques reduce termination and nontermination analysis to safety verification.
One notable approach is the reduction of nontermination to safety, which constructs recurrent sets via inductive invariants~\cite{DBLP:conf/tacas/ChenCFNO14}.
It was introduced in the context of proving nontermination of programs with nested loops, and adapts naturally to the setting of transition systems.

\begin{definition}[Sink states]
    For a Transition System $\tuple{ Init, Tr, x }$, a state $Sink(x)$ is a sink state, if $Sink(x) \land Tr(x,x')$ is unsatisfiable.
    We use $Sink(x)$ to denote the formula describing all sink states in the transition system: 
     $$\forall x. Sink(x) \implies \lnot \exists x'. Tr(x,x')$$
\end{definition}

Sink states can be computed by applying quantifier elimination to $\lnot \exists x'. Tr(x,x')$.
It is possible to construct the following safety problem: $P = \tuple{ Init, Tr, Sink, x }$.
The algorithm for proving nontermination of transition system is as follows:



\begin{algorithm}[t!]
	\SetInd{0.5em}{0.5em}
    \DontPrintSemicolon
    \small
    \newcommand{\Continue}{\textbf{continue}}
    \caption{Nontermination analysis using Safety.}
    \label{Alg:SNA}
    \SetKwInOut{Input}{Input}
    \SetKwInOut{Output}{Output}
    \Input{Safety problem $P = \tuple{Init, Tr, Sink, x}$}
    \Output{NONTERM | UNKNOWN}
    \While{$\top$}{
        $\tuple{res, \phi} \gets safetyCheck(P)$\\
        \Switch{$safetyCheck(P)$}{
            \Case{$\tuple{UNSAFE, trace(x^{(0)}, \dots, x^{(n)})}$} {
                $M \models Init(x^{(0)}) \land trace(x^{(0)}, \dots, x^{(n)}) \land Sink(x^{(n)})$ \label{SNA:UNSAFE}  \\
                $i \gets n-1$ \\
                \While{$i > 0$}{
                    $s_i(x) \gets $ \textit{variable assigment extracted from } $M$ for $x^{(i)}$\\
                    $s_{i+1}(x) \gets $ \textit{variable assigment extracted from } $M$ for $x^{(i+1)}$\\

                    \eIf{SAT ? $s_i(x) \land Tr(x,x') \land \lnot s_{i+1}(x')$}{
                        $Tr(x,x') = Tr(x,x') \land \lnot s_{i+1}(x')$ \label{SNA:BLOCK} \\
                        \break
                    }{
                        $i \gets i - 1$
                    }
                }
                \lIf{i=0} {$Init(x) \gets Init(x) \land \lnot s_0(x)$  \label{SNA:BLOCKINIT}} 
            }
            \Case{$\tuple{SAFE, Inv(x)}$} {
                \label{SNA:SAFE}
                \If{$\forall x. Inv(x) \implies \exists x'. Tr(x,x') \land Inv(x')$} {
                    \Return NONTERM \label{SNA:RECURRENT}
                }
                \Return UNKNOWN
            }
        }
    }
    
\end{algorithm}

The algorithm begins with a safety check of $P$, where the bad states are set as the Sink states.
At line~\ref{SNA:UNSAFE}, the algorithm handles the case when $P$ is unsafe (i.e., a trace reaching a sink state exists).
The safety checker returns a concrete trace $trace(x^{(0)}, \dots, x^{(n)})$, and the algorithm extracts the satisfying model, that allows to calculate the formulas $s_0(x), s_1(x), \dots, s_n(x)$ - representing the reached system state after each step of the trace.
Using these formulas, the algorithm iteratively checks if it is possible to take a transition from $s_i(x)$ to a state, different from $s_{i+1}(x)$.
If it is possible, then the algorithm blocks the state $s_{i+1}(x)$ from the transition (at line~\ref{SNA:BLOCK}), since this state inevitably leads to a sink state.
Otherwise, if such $s_i(x)$ does not exist, then the state $s_0(x)$ is deterministically leading to the sink states, indicating that all executions from $s_0$ terminate; accordingly, $s_0$ is excluded from the initial states.



In the case when $P$ is safe, the algorithm extracts the inductive invariant $Inv(x)$ at line~\ref{SNA:SAFE}.
The algorithm checks whether $Inv(x)$ is a recurrent set at line~\ref{SNA:RECURRENT}, meaning that it is possible to take a transition from every state within the invariant to another state.
If $Inv(x)$ is a recurrent set, then the transition system is nonterminating.
Otherwise, algorithm returns UNKNOWN, as the analysis is inconclusive.
